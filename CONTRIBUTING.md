# Contributing to Virtuoso

## Getting Started
1. Fork the repository
2. Create a new branch: `git checkout -b feat/your-feature`
3. Follow conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `chore:`
4. Create a pull request with detailed description

## Code Standards
- Use meaningful variable names
- Add comments for complex logic
- Test your changes locally
- Keep commits focused and atomic

## PR Guidelines
- Reference related issues using `Closes #123`
- Provide screenshots if UI changes
- Ensure local testing before submitting
- Be patient with reviewers
